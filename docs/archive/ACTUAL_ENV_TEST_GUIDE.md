✅ 开发服务器已启动：`npm run dev`
✅ GLM API Key 已配置
✅ 依赖已重新安装并修复

### 4. 查看计算结果

点击"计算"或"分析"按钮后，您应该看到：

✅ **财务指标**

- NPV（净现值）：负值（合理）

### 5. 生成 AI 投资报告 ⭐

这是本次测试的核心功能！

#### 步骤：

1. **找到报告生成按钮**
   - 位置：在计算结果页面
   - 标签：通常是"生成投资报告"或"生成报告"
   - 图标：📄 或 PDF 图标

2. **点击生成报告**
   - 系统将显示进度条
   - 您应该看到以下进度：
     ```
     [初始化中...] 0%
     [收集中...] 20%
     [AI分析中...] 40%
     [计算中...] 60%
     [生成报告中...] 80%
     [导出中...] 90%
     [完成] 100%
     ```

3. **等待报告生成完成**
   - 完整过程约需 30-60 秒
   - AI 代理将依次执行：
     - 尽职调查
     - 政策分析
     - 技术提案
     - 风险评估
     - 报告叙述生成

4. **查看生成的报告**
   - 报告包含 6 个章节：
     1. 执行摘要
     2. 项目概况
     3. 财务分析
     4. 政策环境
     5. 风险评估
     6. 投资建议

### 6. 下载报告

生成完成后，您可以选择：

✅ **下载 PDF 报告**
- 点击"下载 PDF"按钮
- PDF 文件将保存到您的下载文件夹
- 文件名格式：`investment-report-YYYY-MM-DD.pdf`

✅ **查看 Markdown 源文**
- 点击"查看 Markdown"按钮
- 可以复制或保存原始 Markdown 文本

## 故障排除

### 问题 1：开发服务器无法启动

**症状**：运行 `npm run dev` 时报错

**解决方案**：
```bash
# 清理并重新安装依赖
rm -rf node_modules package-lock.json
npm install

# 重新启动
npm run dev
```

### 问题 2：AI 代理执行失败

**症状**：报告生成卡在"AI分析中..."步骤

**可能原因**：
- GLM API Key 未配置或无效
- 网络连接问题
- API 速率限制

**解决方案**：
1. 检查 `.env` 文件中的 `VITE_GLM_API_KEY`
2. 打开浏览器控制台查看详细错误信息
3. 检查网络连接
4. 如需要，临时禁用 AI 代理（使用模板模式）

### 问题 3：财务指标显示 NaN 或 undefined

**症状**：IRR、NPV 等显示为 NaN 或 undefined

**解决方案**：
- 已通过最近的代码修复解决
- 确保使用最新版本的代码
- 检查表单数据是否完整填写

### 问题 4：PDF 下载失败

**症状**：点击下载按钮后无反应

**解决方案**：
1. 检查浏览器控制台错误信息
2. 确认浏览器允许下载
3. 尝试刷新页面后重新生成报告

### 问题 5：报告内容为空

**症状**：生成的报告章节内容很少或为空

**可能原因**：
- AI 代理返回内容为空
- 模板加载失败

**解决方案**：
1. 查看浏览器控制台日志
2. 检查 AI 代理响应内容
3. 如果 AI 失败，系统应该自动使用模板备用方案
